**function outsside the component**

```jsx
const foo = val => anotherVal => {
  fn(val, anotherVal);
};
```

**Component**

```jsx
const Component = props => {
  const { val } = someHoook();

  return (
    <div>
      <h1>{props.title}</h1>
      <AnotherComponent onFoo={foo(val)} />
    </div>
  );
};
```

I diden't know that you can pass down the first value by callling the function while you sending it as a prop. In this way the _AnotherComponent_ will have access
to it without passing down any extra arguments. This all thanks to the curried function!
A really cool way of doing things, and a functional way of writing your code. If you are intresed I defenitlly recomd to read _Kyle Simpsons book_ [Function light](https://github.com/getify/Functional-Light-JS).
